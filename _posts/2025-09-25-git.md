## 🧩 1. Git 과 Github 차이

### 1-1. Git 의 용도
##### ☑️ Git (Version Controller)
- **코드 변경 사항 추적**  
  예시: [파이널리포트] → [파이널리포트_최종의_최종] → [파이널리포트_최종의_최종_이게_찐_마지막수정본]  
  → Git 은 이런 모든 수정 과정을 체계적으로 저장 가능
- **이전 버전 비교 및 복구 가능**  
  코드가 에러날 경우, 이전 버전으로 복원 가능
- **저장소의 종류**
  - 📂 Local Repo: **내 PC**에 저장
  - ☁️ Remote Repo: **Github 서버**에 저장  
  - 즉, 로컬 ↔ 원격 저장소 연결 구조

---

### 
### 1-2. Github란? 
##### ☑️ Github (Cloud Service)
- Git 저장소를 온라인에 저장 및 관리할 수 있는 서비스
- **장점**
  - 👥 협업 가능
  - 🌍 다른 환경(학교/회사)에서도 동일 환경 작업 가능
  - 📌 개인 프로젝트 관리용으로도 활용 가능

---

### 🔍 Git vs Github

| 구분 | Git | Github |
| --- | --- | --- |
| 역할 | 버전 관리 도구 | Git 저장소 클라우드 서비스 |
| 위치 | 로컬(내 PC) | 원격(서버) |
| 사용법 | `add`, `commit` 등 | `push`, `pull` 등 |

➡️ 로컬에서 코드 작업 → **push** → 원격 업로드  
➡️ 원격 저장소 → **pull** → 로컬 다운로드

### 2-3. 로컬 저장소 초기화
- git 정상 설치 완료되셨으면, 이제 내 컴퓨터에서 **Local repo 설정** 하실 차례
1. VS Code 열고 작업할 폴더 이동 (예: rokey/STUDY)
2. 경로안에 한글 없는거(❌) 확인해주시고~
3. 작업 디렉토리 가셔서 **현재 위치에서 git을 쓰겠다**는 선언을 해야해요.
    - 컴퓨터한테 ‘야 나 여기서 작업하는거 이후에 git에다 저장할거니까 참고해’ 하는 겁니다
        ```bash
        git init
        ```
        👉 .git 폴더 생성되면 성공 ✅

#### 💡(번외) VS Code 터미널에서 Git 인식이 안되는 경우
- ⚠️ Git 다운받고 설치 완료 + 근데 VS Code 터미널에서 git 인식을 못한다?
  - 📌**환경변수 PATH 설정 필수**
    - 이 경우는 환경 변수 PATH 설정이 안되어있어서 VS Code 가 ☹️ 나 git 이 어디에 있는지 모르겠는데? 이슈입니다
  - **해결 방안**
    1. 먼저 VS Code 터미널 말고, Powershell 따로 키셔서 `where git` 입력 → <u>git 설치된 파일 경로</u> 나옵니다. 보통은 둘 중 하나
       - `C:\Program Files\Git\bin\git.exe`
        - `C:\Program Files\Git\cmd\git.exe`
    2. 환경 변수 PATH 에 git 이 설치된 PATH가 등록됬는지 확인하기
        -  **윈도우 검색창** → 환경 변수 검색 → 시스템 환경 변수 편집 열기
        -  오른쪽 아래 환경 변수(N)... 클릭
        -  **시스템 변수**에서 Path 선택 → 편집
        -  거기에 Git 설치 경로`(...\Git\bin)` 또는 `(...\Git\cmd)`가 들어있는지 확인
            - Git 설치 경로가 없으면 **새로 추가**해야 해요. 기존에 있던 다른 PATH 에다가 덮어쓰기 하시면 ㄴㄴ
    3. 환경 변수 PATH 설정까지 다 하셨으면, VS Code 한번 껐다가 켜기(재실행)
        - VS Code 터미널에 `git — version` 해서 깃 버전 정보 확인되면 끝! 👏
    ---
    
### 2-4. 저장소에서 쓸 사용자 정보 설정 : config 설정
- Git을 사용해 **로컬 저장소(Local Repository)** 를 **원격 저장소(Remote Repository)** 와 연결하려면 내 신원정보(`이름, 이메일`)를 미리 설정해두어야 합니다. → *일종의 자동 로그인 개념.*

---

<details>
  <summary> --global vs --local 차이점 </summary>
  </br>

  ##### 🔹 `--global` (전역 설정)
  - 현재 사용자 계정 **전체에 적용**됩니다.  
  - 즉, 내 컴퓨터에서 이 계정으로 사용하는 **모든 Git 저장소**에 동일한 이름과 이메일 주소가 사용됩니다.
  ⚠️ Github 계정이 2개 이상이라면 주의해서 사용하세요!

    - **예시**
      ```bash
        git config --global user.name "Mars"
      ```
  
  ##### 🔹 `--local` (로컬 설정)
  - **현재 디렉토리**(저장소)에만 적용됩니다.
  - 저장소마다 **다른** 이름, 이메일, 환경을 설정하고 싶을 때 사용합니다.

    - **예시**
      ```bash
        # 회사 프로젝트
        git config user.name "회사닉네임"

        # 개인 프로젝트
        git config user.name "개인닉네임"
      ```

  </details>

#### ⚙️ config 설정 방법

<details>
  <summary> 이름(user.name) 설정 </summary>
  </br>
  
  - 나중에 여러사람들끼리 협업할 때, 코드 변경한 사람이 누구인가 **구별**하는데 사용됩니다

  - Github 프로필 이름이랑 달라도 되지만 본인인걸 프로젝트 협업하시는 분들이 **식별할 수 있게** 설정 하시는걸 추천

</details>

<details>
  <summary> 이메일(user.email) 설정 </summary>
  </br>

  - **github 계정 이메일 주소!!**

    ```bash
      git config --global user.name "~~Mars~~ (이름: 원하는대로!)"
      git config --global user.email "~~mars@example.com~~ (이메일: Github 계정 이메일 주소)"
    ```

</details>

---

##### 이름과 이메일 설정 잘 됬는지 확인용 명렁어

> **이름** : `git config user.name` (local 로 저장했을 때 ), `git config —global user.name` (global 일 때)

> **이메일** : `git config user.email`(local), `git config —global user.email` (global)

### 2-5. Github 업로드 : 테스트를 위한 .py 파일 생성

## 🔐 3. 사용자 인증 방법 (코드 Push 하려면 필수) : (1) Token 또는 (2) SSH Key 중 하나만 선택

### 3-0 왜 해야하는가?
- 깃허브가 정책이 바뀌어서 깃허브 계정 로그인 pw로는 이제 `git push` 못해요 → `permission error` 납니다
- 그래서 “**나는 관리자고, 지금 내 코드 Push 할 거임. 방해하지마**” 알려줘야합니다
    - 이때 사용하는게 __**개인 토큰** 또는 **SSH Key**__

### 3-1. Personal Access Token 설정 방법

- Token 은 깃헙에서 받을 수 있어요 → 처음 설정한거 두번 다시 확인 못하니까,  꼭 어딘가에 보관해두셔야 해요
    - 만약 잊어버리면 다시 깃헙가셔서 새로 생성하셔야하는데 이게 좀 귀찮….

🗝️ **Token 생성 방법!** 

1. **Settings → Developer settings → Personal access tokens → classic → Generate new token**
2. **generate new token(classic) > Note/Expiration** 설정 해주시고
3. 옵션들 중에서 
    - [x]  repo
    
    ‘리포’ 체크 후 생성!
    
    → 생성 하면 무슨 이상한 글자들이 나타날텐데 이걸 복사해서 어딘가 보관하셔야해요.

### 3-2. ⭐SSH Key

- 윈도우 terminal 에서 `ssh-keygen`
    
    ```powershell
    ssh-keygen
    ```
    
- **git 튜토리얼**에 사진으로 설명 잘 되어있습니당 → https://github.com/KennethanCeyer/tutorial-git
    - 📌 SSH Key 생성 후 ~/[사용자 폴더]/.ssh/ 폴더에 **`id_xxx.pub` 파일 생성 안됬을 경우!**
        - ssh 다시 삭제 후 새로 생성 → 생성할 때 그냥 따로 설정 안하고 엔터만 치시면 기본값으로 생성되서 `id_xxx.pub` 파일 생성되실겁니다.

- 생성한 키 중 `id_xxx.pub`는 GitHub SSH Key 에 등록 (git 튜토리얼 잘 따라하기👍)
    - `id_xxx**.pub**` 파일만(Public 용)! 그 외에 다른 파일은 **혼자만 보관** → `id_xxx` (개인키)는 절대 공유 ❌

## ☑️ 5. (자주 쓰이는) 주요 Git 명령어 :  Add, Commit, Push, and Pull

### 5-0 init

- `git init` : 나 여기 디렉토리에 작성할 코드들 이제부터 git으로 관리할거야! 선언하기

### 5-1. add

- `git add` : 로컬 저장소에서 수정/추가된 파일을 원격 저장소에 업로드 준비 상태로 만들기
    - `git add .`  : 현재 작업 디렉토리에서 **변경된 모든 파일들을** 준비 상태로 만들고 싶을 때 `.` 사용(*add 와 . 사이에 빈칸 필수*)

### 5-2. commit

- `git commit -m “[commit msg]”` : 업로드 할 코드에 대해서 설명 메세지 작성 → 내가 알아볼 수 있도록 적으시면 됩니다. 영어로 쓸 필요 ㄴㄴ
- 잘 쓰이는 커밋 메세지 예시
    - `Add` (function/module/file): 기능 모듈/함수/파일 등 추가했다.
    - `Modify` (func/update) : 기능 또는 업데이트를 위해 코드 변경했다
    - `Fix`  : 에러 수정 ( syntax 에러, 논리 재구조화, 입출력 변경 등) → (예시 : `git commit -m “Fix: minor typos”`)

### 5-3. Push

- 내 컴퓨터에서 작성한 **코드 파일들 → 원격 저장소로 전송** 할 때 `git push` 씁니다

### 5-4 . Pull (fetch + merge)

- **원격 저장소**에서 새로 업데이트 된 **최신 코드 → 로컬(내 컴퓨터 VS Code)로 가져오기**
- 협업 시, 코드 작업 전에 `git pull` 하는 습관 해주세요.
    - 단!!!!! 버전 충돌 안일어나는지 먼저 확인 하시고!
    - 로컬에서 코드 변경 먼저하고 나중에 Pull 하려면 충돌 일어나서 Pull 안되는 경우가 있슴다

### 5-5. Fetch

- 깃허브 저장소에서 최신 변경 사항을 가져오기만 함 (사실상 변경 사항이 뭔지 가져와 확인만 하는거)

## 👀 6. (몰라도 괜찮을) Git의 파일 버전 관리 시스템 이해하기

### 6-1. 라인별 색상 차이

- 🟩 초록색 라인 = 새로 추가(생성)된 코드라인
- 🟦 파란색 라인 = 변경(수정) 된 코드라인
- 🟥 빨간색 라인 = 삭제된 코드라인

### 6-2. VS Code 파일 디렉토리 옆 U 와 M 의 뜻

- 초록색 **U** (untracked) : git에 현재 해당 코드 파일이 존재하지 않는다. 새로 생성된 파일이다.
- 노란색 **M** (modified) : git에 해당 파일이 존재하지만 달라진게 보인다. 수정된 파일이다.
